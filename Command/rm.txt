* rm [#o47a75d0]


''構文''

&color(blue){①通常};&br;# rm hoge fee bar
#br
&color(blue){②ディレクトリも削除};&br;# rm -r dirname
#br
&color(blue){③強制的に削除};&br;# rm -f 1 2 3 4 free
　　上記削除を実行する際には、仮に削除対象がなくてもエラーは起きない。
　　つまり ''exit code = 0''　で返却される。
　　（通常rmでは、エラーになる / exit code = 1）

#br

**注意事項 [#y5ffcc61]

''削除出来るかどうかは、そのファイルがあるディレクトリ''

rm コマンドでファイルが削除できるかどうかは、本質的にはそのファイルが置かれているディレクトリに
パーミッション「w」(書き込み) が立っているかどうか、で決まる。

COLOR(blue):仮に書き込み権限がないファイル（-r--r--r--）をrmコマンドで削除しようとすると、以下のメッセージが
COLOR(blue):出る。ここでyと入力すると難なく削除出来るし、rm -f ではメッセージすらも出ない。
 rm: remove write-protected 通常の空ファイル `b.sh'?

ただし、その対象ファイルが存在するディレクトリ（つまりカレントのひとつ上でlsを叩いたときの権限）に、
パーミッション「w」(書き込み) が立っていない場合は、rm -f でも当然消せない。
さらに、rm -f でもエラーとなり、exit code = 1が返却される。
